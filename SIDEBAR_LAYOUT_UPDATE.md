# 导航布局更新说明

## 更新概述

已成功实现 Ant Design 风格的两级导航布局：
- 一级菜单：顶部导航栏
- 二级菜单：左侧边栏

## 导航结构

### 一级菜单（顶部）
- 商品管理
- 商品智能匹配
- 数据管理
- 系统管理

### 二级菜单（左侧）
根据当前选中的一级菜单，动态显示对应的二级菜单项。

例如：
- 选择"商品管理" → 左侧显示：商品管理、SPU 列表、SKU 列表、SPU 命名规范检查
- 选择"系统管理" → 左侧显示：账套管理、数据同步、更新日志、操作审计、账套维护时间

## 主要变更

### 1. 组件结构
- `components/Navigation/TopNavbar.tsx` - 顶部导航栏（一级菜单）
- `components/Navigation/Sidebar.tsx` - 左侧边栏（二级菜单）
- `components/Layout/AdminLayout.tsx` - 主布局容器

### 2. 布局特性

#### 顶部导航栏
- 显示 Logo 和系统标题
- 显示所有一级菜单项
- 高亮当前激活的一级菜单
- 右侧显示用户信息

#### 左侧边栏
- 仅在有二级菜单时显示
- 显示当前一级菜单的所有二级菜单项
- 高亮当前页面对应的二级菜单项
- 固定宽度 256px

#### 主内容区
- 包含面包屑导航
- 自适应剩余空间
- 可滚动内容区域

### 3. 设计风格

#### 颜色主题
- 主色调：#1890ff (Ant Design 蓝)
- 激活状态：蓝色高亮背景
- 悬停效果：浅灰色背景

#### 交互体验
- 点击一级菜单自动跳转到该分类的第一个二级页面
- 左侧边栏自动显示对应的二级菜单
- 清晰的视觉层次和状态指示

## 技术实现

### 导航逻辑
```typescript
// 一级菜单点击跳转到第一个子页面
<Link href={item.children?.[0]?.href || '#'}>
  {item.label}
</Link>

// 二级菜单根据当前路由动态显示
if (!parentMenu || !parentMenu.children) {
  return null; // 没有二级菜单时不显示侧边栏
}
```

### 状态管理
- 使用 `useNavigation` hook 获取当前菜单状态
- `parentMenu` - 当前激活的一级菜单
- `currentMenu` - 当前激活的二级菜单（页面）

### 布局结构
```
┌─────────────────────────────────────────┐
│  TopNavbar (一级菜单)                    │
├──────────┬──────────────────────────────┤
│          │                              │
│ Sidebar  │  Main Content                │
│(二级菜单) │  - Breadcrumb                │
│          │  - Page Content              │
│          │                              │
└──────────┴──────────────────────────────┘
```

## 使用说明

### 导航操作
1. **切换一级菜单**：点击顶部导航栏的菜单项
2. **访问二级页面**：点击左侧边栏的菜单项
3. **面包屑导航**：显示当前位置，可快速返回

### 视觉指示
- **当前一级菜单**：顶部蓝色高亮
- **当前二级页面**：左侧蓝色高亮背景
- **悬停效果**：鼠标悬停时显示浅灰色背景

## 优势

1. **清晰的层级结构**：一级菜单在顶部，二级菜单在侧边，层次分明
2. **高效的空间利用**：顶部空间用于主导航，侧边空间用于子导航
3. **符合用户习惯**：类似 Ant Design Pro 等主流后台管理系统的布局
4. **良好的可扩展性**：易于添加新的菜单项和功能模块
