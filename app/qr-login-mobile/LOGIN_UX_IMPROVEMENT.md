# 钉钉扫码登录用户体验改进

## 问题描述

第一次使用钉钉扫码登录时，手机端会显示"需要钉钉登录"的错误提示，但第二次扫码就能正常登录。

## 根本原因

这是一个**时序问题**：

1. 用户扫码后，系统需要调用钉钉 API 进行身份验证（异步过程）
2. 首次登录时，钉钉 API 调用需要 1-3 秒完成
3. 在 API 响应之前，如果出现任何延迟，系统会立即显示错误提示
4. 第二次扫码时，token 已缓存在 localStorage，直接读取，无需等待 API

## 解决方案

### 1. 移动端改进（qr-login-mobile/page.tsx）

- **添加倒计时显示**：扫码后显示 5 秒倒计时，让用户知道系统正在处理
- **延迟错误显示**：只有在倒计时结束后才显示错误信息
- **优化加载状态**：显示"正在通过钉钉登录"和进度提示
- **改进错误提示**：将"需要钉钉登录"改为"登录遇到问题"，提供重试按钮

### 2. PC 端改进（qr-login-desk/page.tsx）

- **增强扫码成功提示**：明确告知用户"请在手机上点击确认登录按钮"
- **优化警告信息**：列出可能导致登录时间较长的原因
- **添加首次登录提示**：说明首次登录需要几秒钟进行身份验证

## 用户体验流程

### 首次登录
1. PC 端显示二维码
2. 用户使用钉钉扫码
3. PC 端显示"已扫码，请在手机上确认"
4. 手机端显示倒计时（5秒），提示"正在通过钉钉登录"
5. 钉钉 API 完成验证（通常 1-3 秒）
6. 手机端显示"确认登录"按钮
7. 用户点击确认，登录成功

### 后续登录
1. PC 端显示二维码
2. 用户使用钉钉扫码
3. 手机端立即显示"确认登录"按钮（从缓存读取 token）
4. 用户点击确认，登录成功

## 技术细节

- 倒计时时长：5 秒
- 倒计时结束后仍未获取到 token，才显示错误
- 使用 `setInterval` 实现倒计时
- 登录成功后立即清除倒计时

## 测试建议

1. 清除浏览器缓存和 localStorage
2. 使用钉钉扫码进行首次登录
3. 观察倒计时和加载提示
4. 验证登录成功后的跳转
5. 再次扫码测试后续登录流程
